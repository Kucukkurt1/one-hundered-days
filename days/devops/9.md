## ğŸ“– Task Explanations

In collaborative development, it's inevitable that multiple people will make changes to the same lines of code. When Git tries to combine these divergent changes, it sometimes can't figure out which changes to keep, leading to a "merge conflict." Today, you'll learn how to identify, understand, and manually resolve these conflicts, a crucial skill for any team member.

You'll also be introduced to `git rebase`, an alternative to `git merge` for integrating changes. While `merge` preserves history, `rebase` rewrites it to create a cleaner, linear commit history. Understanding when to use `merge` versus `rebase` is a key decision in maintaining a project's Git history.

## ğŸ“š Dictionary for Developers

*   **Merge Conflict:** A situation that occurs when Git is unable to automatically combine changes from two different branches because they have modified the same lines in the same file.
*   **`git rebase`:** A command that integrates changes from one branch onto another by moving or combining a sequence of commits to a new base commit.
*   **`git merge`:** A command that integrates changes from a named branch into the current branch, creating a new merge commit.
